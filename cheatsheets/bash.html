CLI Cheat Sheet

Command Prompt (Windows) vs Linux Terminal — common commands side by side.

Section 01

Navigation

Moving around the file system — the most common thing you'll do in any terminal.

Action Windows (cmd) Linux (bash) Notes
Show current directory cd pwd cmd with no args prints path; bash has a dedicated command
Change directory cd foldername cd foldername Same!
Go up one level cd .. cd .. Same!
Go to home directory cd %USERPROFILE% cd ~ Linux shorthand is much tidier
Go to root cd \ cd / Windows uses \; Linux uses /
Remember
Windows paths use backslashes (\Users\name\Documents); Linux paths use forward slashes (/home/name/documents). This trips people up constantly when switching between systems.
Section 02

Listing Files

Seeing what's in the current directory — probably the first command you'll reach for.

Action Windows (cmd) Linux (bash) Notes
List files dir ls The core difference to remember
Detailed list dir /q ls -l Shows permissions, size, dates
Show hidden files dir /a ls -a Linux hidden files start with a dot
Detailed + hidden dir /a /q ls -la Flags combine in Linux: -la = long + all
Pro Tip
In Linux, ls -la is one of the most useful commands you'll use — it shows everything including hidden config files, file sizes, and who owns each file.
Section 03

Files & Folders

Creating, copying, moving, and deleting — handle with care.

Action Windows (cmd) Linux (bash) Notes
Create folder mkdir foldername mkdir foldername Same!
Delete file del filename rm filename rm is permanent — no recycle bin
Delete folder rmdir /s foldername rm -r foldername -r = recursive (includes all contents)
Copy file copy src dest cp src dest Use cp -r for folders in Linux
Move / rename move src dest mv src dest mv doubles as rename in Linux
Create empty file type nul > file.txt touch file.txt Linux touch is much cleaner
Watch Out
rm -r foldername in Linux deletes everything instantly with no confirmation and no undo. There is no recycle bin. Double-check before you press Enter — especially with wildcards like rm -r *.

Renaming a file — before & after

Windows
move oldname.txt newname.txt

The move command handles both moving and renaming.
Linux
mv oldname.txt newname.txt

Same idea — mv moves or renames depending on whether the destination is a folder or filename.
Section 04

Viewing Content

Reading files and searching inside them without opening an editor.

Action Windows (cmd) Linux (bash) Notes
Print file contents type filename cat filename cat can also concatenate multiple files
Page through a file more filename less filename less lets you scroll up and down; q to quit
Search inside a file findstr "text" file grep "text" file grep is far more powerful and supports regex
Search recursively findstr /s "text" * grep -r "text" . Searches all files in all subfolders
Pro Tip
grep is one of Linux's most powerful tools. Combine it with a pipe to search command output: ls -la | grep ".txt" filters the listing to only show .txt files.
Section 05

System & Misc

Everyday utility commands for managing processes, network, and your environment.

Action Windows (cmd) Linux (bash) Notes
Clear screen cls clear Ctrl+L also works in Linux
Print text echo Hello echo Hello Same!
Command history doskey /history history Up arrow scrolls through history in both
Find a program's path where python which python Useful for checking which version is active
Show environment variable echo %PATH% echo $PATH %VAR% in cmd; $VAR in bash
List running processes tasklist ps aux Or use top / htop interactively
Kill a process taskkill /PID 1234 kill 1234 Use kill -9 1234 to force-quit in Linux
Network info ipconfig ip a ifconfig is older; ip a is modern Linux
Ping a host ping google.com ping google.com Same! Linux pings forever — use Ctrl+C to stop
Get help help command man command man = manual; very detailed. Press q to quit
Section 06

Common Gotchas

The things most likely to trip you up when switching between the two systems.

1
Slash direction Windows uses \ in paths; Linux uses /. Scripts written for one will break on the other.
2
Variable syntax Windows wraps variables in percent signs: %VAR%. Bash uses a dollar sign: $VAR. Easy to mix up.
3
rm has no undo Unlike Windows del, Linux rm bypasses the trash entirely. It's gone immediately and permanently.
4
Case sensitivity Linux filenames are case-sensitive: File.txt and file.txt are different files. Windows is not.
5
Ping runs forever Linux ping doesn't stop on its own. Use Ctrl+C to cancel it, or ping -c 4 to limit to 4 packets.
6
dir vs ls The single most common muscle-memory mistake when switching. dir on Linux will just say "command not found".
When to Use
If you're working on Windows but need bash scripting, consider WSL (Windows Subsystem for Linux) — it gives you a full Linux terminal inside Windows, so you don't have to context-switch.
