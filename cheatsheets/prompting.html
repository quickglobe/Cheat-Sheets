Prompting Cheat Sheet

A practical reference for writing effective AI prompts — from first principles to advanced techniques.

Section 01

The Golden Rules

1
Be specific, not vagueAmbiguity forces the model to guess your intent.
2
State the output formatYou get what you describe: list, table, JSON, prose.
3
Context before taskFront-loading context improves comprehension.
4
One instruction per sentenceCompound sentences bury requirements.
5
Show, don't just tellExamples beat abstract descriptions every time.
6
Iterate — don't start overRefine in-thread; the model tracks conversation state.
Section 02

Anatomy of a Great Prompt

A well-structured prompt has up to five layers. Not every prompt needs all five, but knowing the framework helps you diagnose weak prompts.

LayerPurposeExample Fragment
Role / PersonaSets expertise & voice"You are a senior data engineer…"
ContextBackground the model needs"We use PostgreSQL 15 on AWS RDS…"
TaskThe specific action"Write a migration script that…"
FormatShape of the output"Return as a fenced code block"
ConstraintsBoundaries & edge cases"No deprecated syntax; keep < 50 lines"
Section 03

Controlling Tone & Voice

Tone shapes vocabulary, sentence length, and assumed audience knowledge. Specify it early.

TonePrompt Phrase
Casual / conversational"Explain like I'm chatting with a friend"
Professional / corporate"Write in a formal tone suitable for a board memo"
Technical / precise"Use exact terminology; assume a domain expert"
Encouraging / coaching"Be supportive and guide me step-by-step"
Academic / scholarly"Write in third-person with citations where appropriate"
Pro Tip
Combine tone with audience: "Explain OAuth 2.0 in a casual tone for a junior developer who has never implemented auth before."
Section 04

Structure & Formatting

How you format your prompt directly influences how the model formats its answer. Use markdown, XML tags, and whitespace deliberately.

Markdown in Prompts

  • Headers (#, ##) — signal topic hierarchy to the model
  • Bullet / numbered lists — enumerate requirements so none are missed
  • Code fences (```) — demarcate code, data, or example I/O
  • Bold / italic — emphasise key constraints or terms

XML Tags for Separation

Claude responds especially well to XML-style delimiters that separate data from instructions:

<context>…background info…</context> <instructions>…task details…</instructions> <example>…few-shot example…</example>

This prevents the model from confusing your data with your instructions — especially important when your input contains natural language.

Prompt Length

  • Short (<50 words) — fine for simple Q&A or quick tasks
  • Medium (50–200 words) — ideal for most professional tasks
  • Long (200+ words) — necessary for complex, multi-constraint work
Remember
Longer is not automatically better. Each sentence should carry a purpose.
Section 05

Power Techniques

Provide 2–3 input → output examples so the model learns your pattern without lengthy explanation.

Input: "The food was terrible" → Sentiment: Negative Input: "Loved the ambiance!" → Sentiment: Positive Input: "It was okay, nothing special" → Sentiment: ?
When to use
Classification, extraction, formatting transformations — any task where the pattern is easier to show than describe.

Ask the model to reason step-by-step before giving a final answer. Use phrases like "Think through this step by step" or "Show your reasoning before concluding."

When to use
Math/logic problems, multi-step analysis, debugging — any task where the final answer depends on intermediate reasoning.

Telling the model what not to do is often as valuable as telling it what to do.

  • "Do not include any preamble or pleasantries."
  • "Avoid using jargon the user hasn't introduced."
  • "Do not make up statistics; say you're unsure instead."

Assigning a role primes the model's vocabulary, depth, and perspective.

"You are an experienced immigration lawyer in the UK…" "Act as a senior React developer reviewing a pull request…"

Pre-define the skeleton of the answer you want:

"Respond using this structure: ## Summary ## Pros ## Cons ## Recommendation"

Section 07

Common Mistakes

MistakeFix
Vague ask: "Help me with my code"Specify language, error, file, and desired outcome
Over-stuffing: 10 tasks in one promptBreak into sequential, focused prompts
No format spec: "Tell me about X"Add: "in a 3-paragraph summary with headers"
Assuming memory across chatsRe-state critical context or use Memory
Starting over instead of iteratingFollow up in the same conversation thread
ALL CAPS / aggressive toneFirm, clear language works; shouting doesn't help
